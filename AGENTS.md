# Development guide

## Visual QA / Typography rules

1. テキスト同士、画像、隣接セクションの重なりを禁止する。意図的な装飾であっても、本文の可読領域へ侵入させない。
2. 見出しの実寸は所属カラム内に収める。`overflow` で隠す方法をレイアウト不良の解決策にしない。
3. 強制 `<br>` はコピーとして改行を意図した場合だけ使用する。改行後に1〜3文字だけが孤立する状態を作らない。
4. 日本語の折返しは `max-width`、列幅、`font-size`、`gap` を調整して読みやすくする。PCで一行が自然な短文は一行を優先し、スマートフォンでは自然な複数行を許容する。
5. レイアウト変更後は最低1440、1280、1024、768、390pxでスクリーンショットを確認する。横溢れ、孤立行、重なり、CTA欠落を確認する。
6. DOM上で主要見出し・本文の `getBoundingClientRect()` を取得し、異なるカラム間で矩形が交差していないか確認する。DOM検査だけで終えず、画面も目視確認する。
7. 問題をユーザーに指摘されるまで残さず、自律的に余白・改行を調整してから完了報告する。
